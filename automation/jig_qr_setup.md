# 端末治具＋印刷QRの位置合わせ

マウスはQRを読まない。**6タップでスキャナーが開いた瞬間に、カメラが既にQRを映している**ことが成功条件。

## 目標レイアウト

```
          [印刷QR]
             ^
             |  15〜25cm 程度（機種で調整）
             |
      +------+------+
      |   カメラ    |  ← 上面または前面
      |             |
      |   画面      |  ← HIDで中央付近を6タップ
      +-------------+
         治具で固定
```

## 手順

1. `python3 generate_qr.py` で `dist/provisioning_qr.png` を生成
2. **大きめに印刷**（一辺 8cm以上推奨）。ラミネート可だが光沢反射に注意
3. 端末を治具に固定（向き・高さ・傾きが毎回同じになること）
4. QRをカメラの正面にスタンド固定
5. 手動でようこそ → 6タップ → スキャナー起動を確認し、**動かさずに**QRが読めるか見る
6. 読めなければ距離・角度・照明だけ調整（端末位置は極力動かさない）

## チェックリスト

- [ ] 治具に載せたまま電源ON〜ようこそまで画面が見える
- [ ] 6タップ位置が毎回同じ座標で当たる（画面中央が無難）
- [ ] QRスキャナー起動直後、フォーカスなし／最小操作で読取成功
- [ ] Wi-Fi（config の SSID）がその場所で届く
- [ ] GitHub Releases の APK URL に端末から到達できる

## 失敗しやすい点

- ガラス反射・真上の強い照明
- QRが小さすぎる／ピント範囲外
- ケースやホルダーでカメラ穴が半分隠れている
- タップ後に別ダイアログ（言語選択など）が先に出る機種 → HIDシーケンスに追加クリックが必要

## 設定への反映

読取が安定したら `automation_config.json`:

```json
"jig": {
  "qr_ready": true,
  "tap_x": 0.5,
  "tap_y": 0.45,
  "notes": "Pixel7 / QR一辺10cm / カメラから20cm"
}
```

`tap_x` / `tap_y` は画面左上原点の相対座標（0.0〜1.0）。HID側が画面解像度に合わせて変換する。
